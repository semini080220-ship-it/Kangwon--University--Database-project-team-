# 1. 문제 정의 및 기획

## 1.1 지역(Place) 현안
삼척시는 **쏠비치 삼척**, **장호항** 등 일부 대표 관광지에 방문객이 집중된다. 이로 인해
- **병목/혼잡**: 성수기 주차·편의시설 포화, 만족도 저하
- **환경 부담**: 특정 해안·계곡에 답압·쓰레기 등 생태 훼손 집중
- **지역 불균형**: 무건리 이끼폭포, 초곡용굴촛대바위길, 도계 폐광촌 등 **숨은 로컬 명소**는 저평가·저방문

## 1.2 해결 아이디어
관광객을 시간·공간적으로 **분산**시키는 소프트웨어 가이드.
- 실시간 **혼잡도**를 기준으로 붐비는 곳 대신 **한적한 대안/로컬 명소**를 추천
- **친환경/체험형 스탬프 코스**로 분산 동선을 유도
- **친환경 활동 인증**(쓰레기 줍기 등)으로 보존 행동에 보상

## 1.3 소프트웨어 기능 요구사항 (요구사항 → 기능 연계)
| # | 지역 현안 | 기능 요구사항 | 구현 기능 |
|---|---|---|---|
| R1 | 특정지 쏠림 | 혼잡 시 대안 추천 | `GET /attractions/{id}/alternatives` (분산 추천) |
| R2 | 실시간 정보 부재 | 혼잡도 관리 | `PATCH /attractions/{id}/congestion` |
| R3 | 숨은 명소 저방문 | 로컬 명소 노출 | `localGem` 플래그 + 추천 우선순위 |
| R4 | 동선 분산 필요 | 테마별 코스 | 스탬프 코스(Course/CourseDetail) |
| R5 | 보존 동기 부족 | 친환경 활동 인증 | 방문/인증 기록(VisitLog, ECO_ACTIVITY) |
| R6 | 분산지 신뢰 부족 | 후기/평점 | 리뷰(Review), 평균 평점 |

## 1.4 범위
- 본 단계: 관광지(Attraction) 슬라이스 **완성**(CRUD + 혼잡도 + 분산 추천), TDD 검증.
- 후속: Course / VisitLog / Review / User 슬라이스 순차 확장.
